Auth Layer
Why It Matters

Built to be trusted with real data.

Every production app below moves money, moves packages, or moves product — that means auth, payments, and access control had to hold up, not just demo well. This is what's actually implemented.

Dual-token JWT auth
Short-lived access tokens plus a refresh token in an httpOnly cookie — unreachable by injected scripts, so XSS can't steal a session.
Webhook signature verification
Paystack webhooks are verified against a signature before any order is marked paid, blocking forged payment notifications.
bcrypt password hashing
Passwords are never stored in plain text — hashed with bcrypt so accounts stay protected even if a database is ever exposed.
Role-based access control
Admin and customer permissions are enforced server-side, not just hidden behind a UI toggle the client could bypass.
Input validation & CORS
Every request is validated before it touches the database, and CORS rules restrict which origins are allowed to call the API.
Solo end-to-end deployment
Frontend on Vercel, backend on Render — both configured and shipped independently, owning the full path from code to production URL.
